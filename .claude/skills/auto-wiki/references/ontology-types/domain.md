# 领域研究（Domain）采集策略

> 适用于研究一个**领域（domain）**——围绕一个中心实体展开的客观存在域（如「宏观货币政策」「企业年金」「某行业监管」）。典型产出：实体页（机构/工具/指标）、概念页（机制）、事件页、关系网络。
> 本文档是**领域无关的采集引擎规范**。下面用宏观货币政策做例子（央行、7天逆回购、利率走廊、MLF改革……），但规则适用于任何领域。
> **权威契约是各领域自己的 `_ontology.md`**（如 `wiki/macro/_ontology.md`）。本页是引擎侧的通用化说明，遇冲突以领域契约为准。

## 0. 三个总原则（先立规矩）

1. **领域，不是课题**：wiki 按**领域（domain）**组织——`wiki/macro/`、`wiki/年金/` 各是一个独立的本体域，有自己的 `_ontology.md` 和中心实体。**一个研究课题（topic）不建领域**，它降级为该领域 `分析/` 目录下的**一页**（派生视图，删了不影响本体）。
2. **节点 / 数据 / 边 三分**：
   - **数值绝不是节点** —— `7天逆回购利率=1.40%` 不建页，进 `data.db`。图谱里只有「7天逆回购利率」这条命名序列，没有「1.40%」这个点。
   - **关系是边不是点** —— 「央行运行7天逆回购」是一条 `operated_by` 边，不是一个叫「运行」的节点。
   - **分类标签是边不是点** —— 「7天逆回购是价格型工具」里的「价格型」是一条 `classified_as` 边，不建页。
3. **编译是单向的**：`Inbox（人写散文）→ ingest → wiki（本体产物）`。本采集策略只覆盖 **wiki 积累阶段**，把散落的原文结晶进本体；不反向从 wiki 生成 Inbox。

## 采集维度（每条注明落到 6 档时间模型的哪档，见 §时间归位）

### 1. 政策法规 / 制度文件
- **找**：部委规章、行业标准、监管通知、施动主体发布的正式文件
- **提取**：①发布日期+施动主体（→ **事件** T4）②被创设/变更的工具或机制（→ 实体/概念）③核心规则的因果或定义（→ T2 逻辑）④废止/替代关系（→ 退役协议）
- **产出**：`来源/` 页 +（视情况）`事件/` 页 + `工具/`、`机制/` 页 + `relations` 边（`created_by`/`changed_by`/`implements`）

### 2. 领域数据与统计
- **找**：官方统计公报、协会年报、披露口径
- **提取**：①**命名时间序列**本身（如「M2同比」「DR007」）→ 建 `指标/` 实体页，身份是序列②每个时点的**数值**→ 不建页，进 `data_points`（T0）③口径/统计范围说明 → 写进指标页正文
- **产出**：`指标/` 实体页 + `data_points` 行（数值绝不进页面正文当真相源）

### 3. 研究报告与分析判断
- **找**：券商研报、智库报告、学术论文、白皮书
- **提取**：①核心结论与因果链（→ T2 耐用逻辑，写机制页/分析页）②与已有断言的一致/矛盾（→ 触发退役协议或并存标注）③论据数据来源（→ T0/`source_slug`）
- **产出**：`来源/` 页 + `分析/` 页（该研报的派生判断）+ 更新已有实体/概念/机制页

### 4. 案例、实例与事件复盘
- **找**：典型案例、产品方案、改革复盘、判例
- **提取**：①有确定**日期+施动者、发生过就固定**的一次性变化 → **事件** T4②参与方（→ 实体）③时间线节点 → 各自归档为事件，不堆成一段散文
- **产出**：`来源/` 页 + `事件/` 页 + `relations` 边把工具/机制的创设变更挂到事件

### 5. 市场参与者档案
- **找**：机构官网、年报、资质、份额、组织架构
- **提取**：①机构本身（行为主体，有专名）→ `机构/` 实体页②该机构**扮演的角色**（受托人/托管人/做市商……）→ 角色是边或分类标签，**不要和机构混成一页**③本质归属关系 → `relations`（T3 近永久边）
- **产出**：`机构/` 实体页 + `relations`（机构 vs 角色分轴，见禁混规则）

### 6. 历史沿革与制度演进
- **找**：制度演变史、大事记、政策迭代脉络
- **提取**：①每个关键节点的**离散切换事件** → `事件/`（T4）②被切换的**制度态命题**（如「当前政策利率锚=7天逆回购」）→ `facts` 拉链表（T1，靠事件盖章切换）③渐变漂移（如 11 年数量型→价格型）→ 取代理事件打桩，接受建模失真
- **产出**：`事件/` 页 + `facts` 拉链行 + 机制页正文历史化（旧结论不删，加退役 callout）

## 节点类型识别规则 —— individual vs class 的那一刀

### 核心判据（一句话）

> **「我能不能用手指指向'就是这一个'、而且明年用同一个名字还指同一个东西?」**
> 能 → **实体（individual）**；不能、必须先讲一段「它是怎么运作/怎么定义的」才能让人听懂 → **概念（class / 机制）**。

 **「有没有数据 / 有没有结构」会骗你**——利率走廊有上下限结构和数值，但它仍是**概念**（没有「创设它的那一个专名物」，必须先讲机理才能懂）。唯一可靠的判据只有「能不能用手指指向那一个」。**「抽象不抽象 / 有没有数据」不是判据。**

### 节点类型表（领域无关）

| 类型 | 子目录 | 判据 | 宏观例子 |
|---|---|---|---|
| **实体·机构** | `机构/` | 有专名的行为主体 | 中国人民银行、美联储 |
| **实体·工具** | `工具/` | 有专名、被某机构创设并运行的工具/手段 | 7天逆回购、MLF、SLF、降准 |
| **实体·指标** | `指标/` | 可反复取数的**命名时间序列**（身份是序列，不是某天的值） | 7天逆回购利率、LPR、M2、DR007 |
| **概念·机制** | `机制/` | 必须靠内涵定义才能理解的机理/框架 | 利率走廊、利率传导机制、货币政策工具箱 |
| **事件** | `事件/` | 有确定日期+施动者、发生过一次就不再发生 | 2025-03 MLF美式招标改革、2024-10 买断式逆回购推出 |
| **分析** | `分析/` | 基于上述节点得出的研究判断（派生视图，删了不影响本体） | 央行工具箱全景、美联储加息与大类资产 |
| **来源** | `来源/` | 一篇研报/文章原文的承载页，所有断言可溯源到它 | 2026-05-25-某券商固收报告 |

**分类标签不是节点、不建页** —— 「数量型/价格型/结构性」这类用来给一堆东西分桶的维度，是贴在工具上的多维标签（`classified_as` 边），图谱可按它分桶着色，但它本身不是页。

### 分类决策树（ingest 时对每个名词逐项走）

```
对源材料里每个名词:
├─ 是"某指标某时点的数值/口径"吗?       → 不是节点,进 data.db (T0 data_points)
├─ 能用手指指向"就是这一个"吗?
│   ├─ 是行为主体(机构)?                → 机构/
│   ├─ 是被某机构运行的工具/手段?        → 工具/
│   └─ 是能反复取数的命名序列?           → 指标/
├─ 有确定日期+施动者、发生过就固定吗?    → 事件/ (T4)
├─ 是用来给一堆东西分桶的标签吗?        → classified_as 边 (不建页)
├─ 是必须先定义机理才能懂的框架吗?      → 机制/ (概念)
└─ 是我基于以上得出的判断吗?            → 分析/
```

### 禁混规则（Anti-patterns）

用宏观做例子，类似变体存在于任何领域：

| 容易混淆 | 正确做法 | 判断依据 |
|----------|---------|---------|
| **机构 vs 角色** | 机构建实体页；角色是边/分类标签，不建页 | 同一机构可扮演多个角色（央行既是做市商又是监管者）；「角色」指不向「就是这一个」，不是实体 |
| **指标 vs 观测值** | 指标建 `指标/` 页（命名序列）；观测值进 `data_points` | 「7天逆回购利率」是序列实体，「2026-05 该利率=1.40%」是 T0 数值，**数值绝不是节点** |
| **工具 vs 其利率值** | 工具建 `工具/` 页；它的利率是关联的 `指标/` 序列，每期值进 data.db | 「7天逆回购」（工具）≠「7天逆回购利率」（指标序列）≠「1.40%」（T0 值），三层分清 |
| **分类标签 vs 实例** | 分类标签是 `classified_as` 边，不建页；实例建实体页 | 「价格型/数量型」是分桶维度（边），「7天逆回购」是被分桶的工具（实体） |
| **概念结构 vs 概念是节点** | 利率走廊建 `机制/` 概念页，其上下限用 `bounds` 边画结构 | 走廊有数值有结构，但仍是概念；它的上/下/中枢由 `指标 →(bound_role) 机制` 的 `bounds` 边连接，而非把数值写进页 |
| **事件 vs 状态** | 事件建 `事件/` 页（一次性、永不回退）；状态进 `facts` 拉链表（一段时间为真、被事件切换） | 「2025-03 MLF改革」是 T4 事件；「当前政策利率锚=7天逆回购」是 T1 状态，靠该事件盖章切换 |

**新建 vs 更新**：先搜领域 hub 页面与各页 `aliases:` 字段（OMO=7天逆回购、MDS=买断式逆回购），已有页面就更新，**不要新建同义页面**。

## 采集维度按 6 档时间模型归位（提取后，每条知识落到哪张表）

每条采集到的知识都要定一个时间档，决定它记在哪：

| 档 | 是什么 | 采集时的信号 | 落到哪 |
|---|---|---|---|
| **T0 观测值** | 某指标某时点的一次测量 | 「换个 period 就是新记录」 | `data_points`（值是数字，补 `recorded_at`/`source_slug`） |
| **T1 状态/制度态** | 一段恒为真、被事件切换的命题 | 「会被一个有日期的事件切换」 | `facts` 拉链表（object 是文本/slug） |
| **T2 耐用逻辑** | 跨周期的因果链/定义，可证伪 | 「是 if→then、能被结构变化证伪」 | 机制页正文 + `durability`/`preconditions`/`falsifiable_by` 字段（也可入 `facts`） |
| **T3 实体/近永久关系** | 客观存在物及本质归属 | 「有专名可查询、或本质归属边」 | 实体页 + `relations`（带 `valid_from`/`valid_to`） |
| **T4 事件** | 有日期+施动者、发生即固定 | 「发生过就不再改」 | `events` 表 + `事件/` 页，**只增不改** |
| **T5 类型公理** | 约束「别的知识长什么样」 | 「管的是一整类知识的形状」 | 领域 `_ontology.md` + DB CHECK 约束 |

> **关键裁决**：「当前政策利率=7天逆回购」是 **T1 状态**（`facts`，object 是文本，靠事件切换）；「7天逆回购利率=1.40%」是 **T0 测量**（`data_points`，值是数字，逐月一条）。两种物，记录方式必须不同。

**记录三铁律**（ingest 时必须遵守）：
1. **退役不删除**：T1/T2/T3 任何「变化」都是「旧行封 `valid_to` + 插新行」，**永不 DELETE**。覆盖原值只允许在 T0 同 period 纠错。
2. **双时态分轴**：每条断言记两条时间轴——`valid_from/valid_to`（世界上何时为真）+ `recorded_at`（我哪份研报记的）。`valid_from` 变=世界变了；只动 `recorded_at`=我自己勘误。绝不能混。
3. **事件盖章**：一切 T1/T2「当前态」的改变，都要能追溯到一个 T4 事件（`caused_by_event`）。

## 受控关系词表（关系是一等公民，但仍是边）

关系 `type` **不允许自由文本**，必须从受控词表选。每条边有 `source→target` 类型约束，`lint` 拒绝越界边。

| type | source → target | 含义 | 例 |
|---|---|---|---|
| `operated_by` | 工具 → 机构 | 工具由谁运行 | 7天逆回购 → 中国人民银行 |
| `implements` | 机构 → 机制 | 机构实施某机制 | 中国人民银行 → 利率走廊 |
| `transmits_to` | 指标/工具 → 指标 | 传导（有向） | 7天逆回购利率 → DR007 |
| `bounds` | 指标 → 机制（边属性 `bound_role=upper/lower/center`） | 构成走廊上/下限/中枢 | SLF利率 →(upper) 利率走廊 |
| `classified_as` | 工具 → 分类标签 | 贴分类维度（可多维并列，标签不建页） | 7天逆回购 → 价格型 |
| `instance_of` | 工具/指标 → 机制 | 个体属于某类 | MLF → 货币政策工具箱 |
| `part_of` | 机制 → 机制 | 机制内部包含 | 利率走廊 → 利率传导机制 |
| `created_by` / `changed_by` | 工具/机制 → 事件 | 个体的创设/变更挂到事件 | MLF利率锚地位 →(changed_by) 2025-03-MLF改革 |
| `references` | 分析/来源 → 任意 | 引用（只溯源，不进语义图） | 央行工具箱全景 → 7天逆回购 |

**关系双写（三处一致）**：页面 `frontmatter.relations[]`（给 Obsidian 连边）+ `data.db relations` 表（给查询/校验，带 `valid_from`/`valid_to`）+ 正文里 `[[wikilink]]` 自然引用。

## 命名与目录纪律

- **单一中文 slug = 文件名 = `[[wikilink]]` = `data.db` 主键**。**不**另起 PascalCase 英文 id（会造出第二套命名，和 Obsidian 的中文文件名链接打架）。跨研报去重靠 `aliases:` 字段。
- **wiki/ 是可见目录**：纳入 git、进 Obsidian 图谱。**禁止 `.wiki/` 点目录**（Obsidian 隐藏 dotfolder，图谱里看不见）。
- **类型即目录，目录即图谱着色**：顶层目录 ≤ 6 个（≤ 6 色，9 色肉眼分不清）。

## 页面结构模板

### 实体页（机构 / 工具 / 指标）

Sections：**概况**（一句话定位，能用手指指向的那一个）→ **关键属性**（工具的期限/利率、指标的口径与时点，注明来源；数值真相源在 data.db，正文只摘录）→ **关系**（wikilink + frontmatter `relations` 到关联实体/概念）→ **历史**（时间线指向 `事件/` 页，过时状态历史化不删）

frontmatter 关键字段：`type: entity` + `subtype: institution|instrument|indicator` + `aliases` + `relations`。

### 概念页（机制）

Sections：**定义**（一段话，注明来源）→ **机理**（怎么运作，这是它必须靠内涵定义的原因）→ **结构**（如走廊上下限，用 `bounds` 边而非把数值塞进正文）→ **耐用度与失效条件**（T2：`durability` + `preconditions` + `falsifiable_by`）→ **易混辨析**（与 [[相近概念]] 的区别）

frontmatter 关键字段：`type: concept` + `durability: high|medium|low` + `preconditions` + `falsifiable_by` + `relations`。

### 事件页

Sections：**事件**（一句话：什么时间、谁做了什么）→ **影响**（创设/变更了哪些工具/机制/状态）→ **退役声明**（`retires:` 旧命题、`sets:` 新状态）

frontmatter 关键字段：`type: event` + `event_date` + `actor` + `retires` + `sets`。事件页 **append-only，建后不改**。

### 分析页

一篇研报或一个研究课题的**派生判断**。Sections：**结论** → **论据**（`references` 到实体/概念/事件）→ **与他源的一致/矛盾**。删了不影响本体。

### 来源页

Sections：**基本信息**（发布机构/日期/文号）→ **核心内容**（3-5 条要点）→ **涉及节点**（wikilink 列表，便于 ingest 定位更新目标）

所有页面 frontmatter 遵循 wiki-format.md（title / type / created / updated / sources / confidence），并以领域 `_ontology.md` 的模板为准。
