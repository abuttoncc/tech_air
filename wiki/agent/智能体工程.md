## 技术 (4)
- [[LangChain]] — LLM 应用基础框架与生态，提供模型抽象与中间件机制（整栈底座）
- [[LangGraph]] — 把 LLM agent 建模为有向图状态机的编排框架，整个 Agent 的底座
- [[deepagents]] — 构建在 LangGraph 上的 agent harness（planning/filesystem/subagents/memory）
- [[Palantir 本体]] — Palantir AIP/Foundry 运营层，决策中心架构的工业级实现样板（OAG 母本）

## 原理 (4)
- [[状态化执行]] — 序列化持久化执行状态，使多轮/中断恢复/跨重启续跑（T2 定义性逻辑）
- [[推理内容回传]] — thinking/reasoning 跨轮回传的独立生命周期：结构隔离 + 按签名显式处理（war story）
- [[检索增强生成]] — RAG：生成前检索文档片段注入上下文，数据中心、只接「数据」一类原语
- [[本体增强生成]] — OAG：把生成锚定在本体上，经工具范式同时接数据/逻辑/行动（超越 RAG）

## 模式 (9)
- [[协议桥接管道]] — 多源 agent 事件流归一为单一客户端协议的三段管道（Source→Middleware→SSE）
- [[中间件分层编排]] — 把 agent 能力组织成可插拔中间件栈（基础设施型/功能型双层 + 声明式注册表）
- [[供应商中立路由]] — 多 LLM 供应商藏在统一接口后：选型/协议适配/降级/用量归一
- [[决策中心架构]] — 本体建模对象是「决策」非数据；Data/Logic/Action 三柱合一（OAG 题眼）
- [[工具调用编排]] — LLM 当指挥家拆意图，精确计算交确定性工具（tools paradigm）
- [[逻辑绑定]] — logic binding：异构/跨环境逻辑资产统一成可调接口
- [[沙盒情景]] — Scenario：拟议决策先落本体沙盒子集推演，确认再 commit
- [[写回模式]] — Writeback：安全写回 + 三权分立（探索/暂存/提交）+ 自由度可放可收
- [[人在回路]] — Human-in-the-Loop：人是最后的闸，copilot 信任随表现渐放权
