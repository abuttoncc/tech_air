# 检索增强生成

在 LLM 生成前，先从外部语料检索相关片段注入上下文，让回答锚定在可溯源的资料上，而非只靠参数记忆。这是缓解幻觉与知识时效两大痛点的主流范式。

它的局限在于**数据中心**：检索的对象是文档/片段，接住的只有「数据」这一类原语，接不住逻辑（怎么算）和行动（怎么落）。[[本体增强生成]] 正是冲着这个天花板去的——把检索对象从文档堆换成有对象、关系、状态的本体，并让 LLM 经工具范式同时够到数据、逻辑、行动三类原语。
