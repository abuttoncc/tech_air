# 本体增强生成

把 LLM 的生成锚定在**本体**上的范式：检索、推理、行动都通过本体里的对象、关系、状态进行，而不是只在文档片段里捞数据。它是 [[检索增强生成]] 的演进——RAG 接住「数据」，OAG 经可扩展的工具范式让 LLM 同时接住**数据 / 逻辑 / 行动**三类原语，越过 RAG 只接数据的天花板。

它是 [[决策中心架构]] 的认知引擎：LLM 一绕过本体自己拍脑袋，幻觉就回来；每条建议都要能溯回本体里的具体对象——这是 OAG 跟普通 RAG 拉开差距的根本。
