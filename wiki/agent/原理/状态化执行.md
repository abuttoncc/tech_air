# 状态化执行

智能体执行的一种基本范式：把一次会话的执行状态（conversation、thread、checkpoint、sandbox 上下文）**序列化并持久化**，而不是只活在单次请求的内存里。它使三件事成为可能——多轮对话沿同一线程累积、中断后可从断点恢复（HITL / 断连重连）、进程重启后状态不丢。

这是一条**定义性的耐用逻辑**（durability: high）：只要 agent 仍以「跨轮持续工作」为目标，它就成立；只有当系统退回无状态单轮问答时才被证伪。

在 ablemind 里，状态化执行由 [[LangGraph]] 的 checkpointer / store 机制实现（图在可序列化 state 上运行），落盘到 [[PostgreSQL]]。这正是「持续型研究助手」区别于「会聊天的模型」的工程基础——任务接得住、过程能追溯、产物挂在同一条会话链上。
