# 中间件分层编排

智能体把横切能力组织成**可插拔中间件栈**的架构模式。中间件包裹在模型/工具调用周围（基于 [[LangChain]] 的 wrap 钩子），整条链的顺序与构成由配置决定。两个关键维度——

**① 分层（dual-layer）**：

- **基础设施型**——管健壮性：模型重试 / 工具重试 / 模型降级 / 调用上限 / 模型选择（按轮换模型）/ 上下文压缩。
- **功能型**——加能力：用户记忆 / 技能过滤 / 工具结果预算 / MCP 大结果缓存 / PII 脱敏 / 稳定性观察与锚点注入。

[[deepagents]] 还会自动注入一组基础中间件（规划 / 记忆 / 技能 / 文件系统），本地再叠加项目特有的若干层。

**② 声明式装配（registry）**：yaml 声明「顺序 + 装哪些」，工厂函数（惰性导入）负责「怎么构造」——每个工厂吃 `MiddlewareBuildContext` + 配置，返回 1 个 / 一组 / None。这样改 agent 行为只需动配置，不必在 Python 里硬接链条；顺序本身是有契约的（chain-order spec）。

它教的道理：**agent 的能力是「装配出来的」而非「写死的」**——把健壮性与功能解耦成独立中间件，按声明顺序组合，可独立增删、独立测试、按场景重排。
