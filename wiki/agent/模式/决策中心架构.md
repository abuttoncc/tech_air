# 决策中心架构

本体建模的对象是**决策**，不是数据。每个组织的头等使命是在内外条件不停变时做出尽可能好的决策；传统数据/分析架构只管报表，不捕获决策背后的推理、也不捕获决策落地的动作，所以学不会、也接不进 AI。

一个决策拆成三要素，本体把三者合进一个实时演化的统一模型：

- **数据 Data**——做决策的信息（名词 / 语义层 semantic：对象与关系）。引擎是 [[本体增强生成]]。
- **逻辑 Logic**——评估决策的过程（异构逻辑资产经 [[逻辑绑定]] 统一成工具，由 [[工具调用编排]] 调度）。
- **行动 Action**——决策的执行（动词 / 动能层 kinetic：[[写回模式]] 安全落地，[[沙盒情景]] 先彩排，[[人在回路]] 守闸）。

「名词 + 动词凑成完整句子」：光统一数据、光拼齐逻辑都有价值，但只要执行出来的决策没跟运营系统同步，价值终归有限。三柱齐 + [[决策血缘]] 留痕，才闭合成能持续学习的决策平台。
