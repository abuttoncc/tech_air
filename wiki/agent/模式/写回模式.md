# 写回模式

把决策**推回真实世界**（事务系统、ERP、边缘设备、应用）。闭合行动回路是运营系统跟分析系统的分水岭——数据是名词、逻辑是评估，行动才让决策落地；只要执行出来的决策没跟运营系统同步，前两柱的价值终归有限。

工程要点：

- **三权分立**：能**探索**决策的人 / 能**暂存待审**的人 / 能**提交落地**的人，三种权限分开，细到单字段。
- **按系统调写回**：仓储走 API 更新、ERP 走原生连接器（守各自护栏）、生产计划收平文件异步消化——写回例程一系统一调，不是一把梭。
- **护栏 = 给 AI 划可行边界**：跟数据、逻辑同一套权限校验。默认 AI 只能把动作暂存成 [[沙盒情景]] 交 [[人在回路]] 终审；跑熟的流程可获准自动闭环。**给 AI 的自由度可放可收，改了立刻全库生效。**
