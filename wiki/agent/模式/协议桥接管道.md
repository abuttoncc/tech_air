# 协议桥接管道（Protocol Bridge Pipeline）

一种**流式协议归一**的架构模式：把多种异构的 agent 事件源（[[LangGraph]] 本地图、远程 agent server、gRPC 等）统一适配成一个稳定的客户端协议，并在中间插入可插拔的处理层。三段结构——

```
Source Adapter（源适配）→ Stream Middleware（中间件富化）→ SSE Writer（协议输出）
```

- **Source Adapter**：把某一源的原生事件流翻译成内部统一事件（text / reasoning / tool-call / finish / error / data-channel）；新源只写一个 adapter 即可接入。
- **Stream Middleware**：对统一事件流做富化与旁路——状态富化、源标记、run-id 标注，`PersistenceTap` 把事件旁路写进 [[PostgreSQL]]（message_events），`DeliveryTap` 把 agent 提到的文件挂成附件。
- **SSE Writer**：序列化为 [[AI SDK v6]] 兼容的 [[SSE]] 协议，带断连 keep-alive 与缓冲。

它教的核心工程道理：**所有对客户端的输出必经同一条管道**，绝不在路由层手写 SSE 字符串——这样事件顺序、生命周期、持久化、断连恢复都能在管道内部统一保证，而不是事后打补丁。其行为契约被结晶成可执行不变量（FinishEvent 必须最后、ErrorEvent 不得在 Finish 之后、`data-*` 必带类型判别符、reasoning 先于 text）。

## 关联

- **输出协议（跨域 → web）**：[[AI SDK v6]]（built_on）
- **持久化旁路（跨域 → data）**：[[PostgreSQL]]（built_on，PersistenceTap 写 message_events）
- **主要事件源**：[[LangGraph]]（LangGraphAdapter 消费其事件流）
