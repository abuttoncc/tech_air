# 供应商中立路由

把多家 LLM 供应商（Qwen/DashScope、DeepSeek、GLM、Claude、OpenAI、Ollama…）藏在**一个统一接口**背后的网关模式。上层只声明「要哪个 tier 的模型」，路由层负责：选型（配置 SSOT，如 models.yaml）、协议适配（OpenAI 格式 ↔ Anthropic 格式的消息互转、tool_call_id 客户端合成、usage 字段归一）、跨供应商**降级**（429 / 超时 / 无有效 key → fallback 到备用供应商）、以及 key 池的 sticky 分流。

在 ablemind 里它以 LangChain ChatModel 的形态对外（`RouterChatModel`），因此对 [[中间件分层编排]] 里的 model_selector 中间件透明——换模型只是改 `AgentContext.model_name`。它教的道理：**模型供应商应是可替换的底层依赖，而非写死的耦合**；统一接口 + 降级 + 用量归一，让上层不必关心"今天用哪家"。

其中最难的一块——历史推理内容如何跨轮回传——单独抽成 [[推理内容回传]]。
